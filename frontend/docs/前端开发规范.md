# 前端开发规范

> 适用范围：AI 测试平台前端（`frontend/`，Vue 3 + Vite + TypeScript + Element Plus + Pinia）
> 目的：统一写法与工程约定，让新人和 AI 辅助工具开箱即符合团队标准，降低沟通与返工成本
> 版本：v1.0（2026-07）｜姊妹篇：[《前端 UI 设计与优化规范》](./前端UI设计规范.md)、[《技术方案选型说明》](./技术方案选型说明.md)

---

## 0. TL;DR（最少记住这几条）

1. **包管理只用 pnpm**；`src/` 只写 TypeScript，**禁止新增 `.js` 源文件**；Vue SFC 统一 `<script setup lang="ts">`。
2. **提交前自动过钩子**：pre-commit 跑 lint+format，commit-msg 校验中文 conventional commit，pre-push 跑 `vue-tsc` 类型检查——**类型基线必须全绿**。
3. **类型不手写、不用 `any`/`unknown`**：API 实体类型一律用后端自动生成的 `Schemas['XxxOut']`。
4. **API 调用只从 `@/api` 具名导入**；样式只用 `--ax-*` design token，不写死颜色/尺寸。
5. **组件体量红线 300 行**（新代码强制，存量超限文件为技术债不强改）。
6. **不提交**：`node_modules`、`dist`、`vite.config.ts` 里的本地 proxy target 改动。

## 1. 技术栈与工具链

| 项 | 选型 |
| --- | --- |
| 框架 | Vue 3（`<script setup>`）+ Vite 6 |
| 语言 | TypeScript（`strict`，`allowJs` 渐进兼容，已基本迁移完成） |
| UI | Element Plus（重组件）+ Tailwind v4 / shadcn-vue（布局与门面页），共享 `--ax-*` token |
| 状态 | Pinia ｜ 路由 vue-router ｜ 编辑器 Monaco（本地 worker，无 CDN） |
| 包管理 | **pnpm**（`packageManager` 锁版本；`preinstall: only-allow pnpm` 拦截 npm/yarn） |
| 质量 | ESLint 9（flat config）+ Prettier + husky/lint-staged/commitlint + `vue-tsc` |

常用脚本（`frontend/`）：
```bash
pnpm dev            # 开发服务器 :5173
pnpm build          # 生产构建 dist/
pnpm lint           # eslint src（lint:fix 自动修）
pnpm format         # prettier 格式化 src
pnpm typecheck      # vue-tsc --noEmit，必须 0 error
pnpm gen:api-types  # 从 backend/openapi.json 生成 src/api/schema.d.ts
```

## 2. 目录结构

```
src/
  api/          # 数据访问层：request.ts + 按域模块 + index.ts 桶式再导出 + schema.d.ts(生成)
  assets/ styles/  # 全局样式、design token
  components/    # 复用组件（apifox/ 下为接口自动化模块专用）
  composables/   # 组合式函数（use*.ts）
  config/        # 菜单、apifox 等静态配置
  constants/     # 常量（如 pagination.ts）
  layouts/       # 布局外壳（MainLayout）
  router/        # 路由 + 守卫
  stores/        # Pinia
  types/         # 前端 UI 类型（apifox.ts 等）
  utils/         # 纯函数工具
  views/         # 页面（apifox/ 下为工作台与项目工作区）
```

## 3. 命名规范

- **组件/SFC 文件**：PascalCase（`ProjectWorkspace.vue`、`PageCard.vue`）。
- **composable**：`useXxx.ts`（`useSaveConflict.ts`）。
- **工具/常量/类型文件**：camelCase（`apifoxSpec.ts`）；常量值 `UPPER_SNAKE_CASE`。
- **变量/函数** camelCase；**类型/接口** PascalCase；布尔量用 `is/has/can` 前缀。
- 中文注释、中文文档、中文 commit message。

## 4. Vue SFC 规范

- 统一 `<script setup lang="ts">`；SFC 标签顺序沿用存量：**template → script → style**。
- **props/emits 用泛型声明**：`const props = defineProps<{...}>()`、`defineEmits<{...}>()`；需要默认值用 `withDefaults`。script 中引用 props 必须接 `const props =`，template 可直接用 prop 名。
- 样式一律 `scoped`，穿透用 `:deep()`。
- `v-for` 必带 `:key`，且**不要用会变动列表的数组下标**作 key。
- 列表页/表格布局遵循 [《前端 UI 设计规范》](./前端UI设计规范.md)（`PageCard`、表格 `fill` 一屏化、token 等）。

## 5. TypeScript 规范

- `src/` **只写 `.ts` / `.vue(lang=ts)`**，不新增 `.js` 源文件；根构建配置也已 TS 化（`vite.config.ts`）。
- **类型从后端推导，不手写、不用 `any`/`unknown`**：
  - API 实体统一用生成类型 `Schemas['XxxOut']` / `Schemas['XxxCreate']`（`@/api/types`）；`schema.d.ts` 由 `pnpm gen:api-types` 从后端 OpenAPI 生成，**不手工维护**。
  - 后端接口调整后重跑 `pnpm gen:api-types` 更新类型。
- 路由动态参数（如 `projectId`）统一用 `useRouteParamId()`（`composables/useRouteParamId.ts`），**禁止直接解构 `route.params`**。
- 前端专用 UI 类型放 `types/`（如 `types/apifox.ts`），**不要从 `.vue` 文件 `export type`**。
- 类型基线：`pnpm typecheck` 必须 0 error（pre-push 钩子强制）。

## 6. 组件规范

- **单组件体量红线 300 行**：新组件强制，超了就拆（子组件 / composable 抽逻辑）。存量超限的大文件（如旧 `ApiAutomation.vue`）列为技术债，不在本次强改。
- 复用优先：先找现有组件/composable/工具，避免重复造。
- apifox 前端：`views/apifox/ProjectWorkspace.vue` 是项目工作区外壳（顶部 tab 切 `views/apifox/sections/` 各面板），复用组件在 `components/apifox/`。

## 7. 样式规范

- **双轨**：Element Plus 管重组件（表格/表单/弹窗/树），Tailwind + shadcn-vue 管布局与门面页；两轨共享 `src/styles/tokens.css` 的 `--ax-*` token。
- **只用 token**：颜色/圆角/间距/阴影一律 `var(--ax-*)`，**禁止硬编码 hex / rgba**（需透明度用 `color-mix`）。
- Tailwind 工具类**不放进 `@layer`**（全局有 `* {margin:0}` 重置，入 layer 会被盖掉）；不引入 preflight（避免破坏 EP）。
- 详见 [《前端 UI 设计规范》](./前端UI设计规范.md)。

## 8. API 层规范

- **只从 `@/api` 具名导入**（`import { projectApi } from '@/api'`）；`index.ts` 是桶式再导出，按域模块拆在 `api/*.ts`。
- axios 单实例在 `request.ts`：baseURL `/api/v1`、自动带 token、401 跳登录、响应拦截器直接返回 `response.data`。
- **`/apifox/` 路径的 409 不弹全局错误**，由调用方配合 `composables/useSaveConflict.ts` 处理保存冲突（apifox 六实体带乐观锁 version）。
- SSE 流式（执行引擎/AI 生成）走 `request.ts` 的公共封装。
- baseURL 走 Vite proxy；**联调改 `vite.config.ts` 的 proxy target 后不要提交**。

## 9. 状态管理（Pinia）

- stores：`user`（登录态/权限）、`workspace`（当前项目）、`apiTabs`、`aiGenerate`。
- 组件内不直接存后端可派生的状态；跨页共享才进 store。

## 10. 路由与权限

- 路由守卫在 `router/index.ts`，每个路由 `meta.permission` 对应后端菜单权限（`app/constants/menus.py` ↔ 前端 `config/menus.ts`，两边要对齐）。

## 11. Git 与提交规范

- 分支流：本地功能分支 → PR 到 `test` → `test` 合入 `main`。
- **中文 conventional commits**：`feat(apifox): …`、`fix(frontend): …`、`refactor/style/chore/docs(...)`。
- 钩子（`.husky/`）：pre-commit（lint-staged：eslint --fix + prettier，仅 `src/`）、commit-msg（commitlint）、pre-push（`vue-tsc`）。
- **前端 PR 只提交源码 + package.json**，不含 `node_modules` / `dist`（node_modules 历史被跟踪，其变化不要提交）。

## 12. 本项目适配说明（与通用规范的差异）

- **SFC 标签顺序**：沿用存量 template→script→style，不做全量翻转。
- **组件 300 行红线**：仅对新代码强制；存量超限大文件（旧接口自动化模块等）列技术债，暂不动。
- **目录**：`constants/`、`types/` 已补齐；`directives/` 等按需增补。
- **API baseURL**：走 Vite proxy 而非 `VITE_API_BASE_URL`（保持现状）。
- **两套接口自动化并存**：旧 `api_automation` 不删除，新 `apifox` 为开发重心与范式；改动勿误伤旧模块。
