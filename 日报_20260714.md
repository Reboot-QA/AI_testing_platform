## 里程碑

- **接口 Body 补全收官**：body 类型从 none/json/form-data/urlencoded/raw 补齐到 **8 种全 Apifox 对齐**（+xml/graphql/binary）。
- **并发编辑冲突检测六实体齐**：endpoint / case / scenario / script / schema / dataset。
- **数据驱动整场景闭环**：可配置（P-C）+ 可读（分组展示）。

## 评审拦截的关键问题（均已整改）

- **Body 补全**：pydantic 契约层漏加字段 → cookies/graphql 被静默丢弃、前端填的存不下发不出。教训：单元测 build_request 不够，必须覆盖 pydantic 契约层；已补「删字段即失败」的契约保真回归。
- **P-C**：数据集删除保护漏算场景 run_config 绑定（可误删被依赖数据集）；迭代中途异常卡 running（并发硬规则）。
- **Binary**：上传按 Content-Length 提前拒绝超大体、projectId 改显式 prop 对齐既有约定。

## 技术决策

- **Binary 存储**：选 DB blob 而非文件系统 —— 容器/多实例零卷配置、随 DB 备份、随项目删除级联、纯新表零迁移（用户拍板）。
- **数据驱动分组**：按 `step.iteration` **字段过滤**而非位置切片，因场景含 if/loop 时各轮步骤数不固定（专门补回归钉死）。
